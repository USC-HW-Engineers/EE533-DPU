# EE 533: Lab 6 Quad HW-Threaded ARM ISA Processor Synthesis & Integration Reference

## Project Identification
*   **Design Name**: ARM_Processor_4T
*   **Target Device**: Virtex-2 Pro (NetFPGA v2.1)
*   **Toolchain**: Xilinx ISE 10.1

## Hardware Interface (ARM_Processor_4T.v)
The top-level core has been modified to support external monitoring and memory access via the NetFPGA register interface.

### New External Ports:
*   `EXT_PC_OUT [10:2]`: Outputs the current Program Counter.
*   `EXT_DM_DIN [63:0]`: Input data for external DMEM writes.
*   `EXT_DM_ADDR [7:0]`: Address index for external DMEM access.
*   `EXT_DM_WEN`: Write Enable for external DMEM access.
*   `EXT_DM_DATA [63:0]`: Output data for external DMEM reads.

### Internal Logic Updates:
*   **DMEM Muxing**: Implemented hardware muxes to select between Internal Pipeline access and External Register access for Data Memory (BRAM).
*   **PC Monitoring**: Connected the IF-stage PC to the external output port.

## Software Register Mapping (NetFPGA/cpureg2)
Base Address: `0x02000200`

| Register Name | Offset | Hardware Connection | Purpose |
|---------------|--------|---------------------|---------|
| REG_IMEM_ADDR | 0x00   | `imem_addr`         | Instruction Memory Address (Byte Aligned) |
| REG_IMEM_DATA | 0x04   | `imem_data`         | Instruction Memory Data |
| REG_DMEM_ADDR | 0x08   | `dmem_addr`         | Data Memory Index (0-255) |
| REG_DMEM_DATA_WR | 0x0C | `dmem_data`        | Data Memory Write Input (32-bit, padded to 64) |
| REG_CPU_CTRL  | 0x10   | `cpu_ctrl`          | bit0: RESET, bit1: IMEM_WE, bit2: DMEM_WE |
| REG_CURRENT_PC | 0x14  | `internal_pc`       | Program Counter Monitor |
| REG_DMEM_DATA_LOW_RD | 0x18 | `dmem_data_rd_lo` | DMEM Read Output (Bits 31:0) |
| REG_DMEM_DATA_HIGH_RD | 0x1C | `dmem_data_rd_hi` | DMEM Read Output (Bits 63:32) |
| REG_CYCLE_COUNT | 0x20 | `cycle_counts`      | Total Execution Cycles |

## Address Mapping Summary
*   **Instruction Memory**: Uses byte-addressing in software (`addr * 4`) to map to 32-bit word indices in BRAM.
*   **Data Memory**: Uses direct word indices (0-255) via the register interface. CPU accesses this same memory using byte addresses `0x800 - 0xBFF`.
